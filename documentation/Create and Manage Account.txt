Use Cases with Flows and Alternative Flows:
Use Case 1: Create and Manage Account
Actor: User
Pre-condition: The user has not created an account yet.
Description: A user can create an account by entering details like username and password. Once created, they can manage their profile, update their personal information, and change their password as either freelancer or employer.
Basic Flow:
The user navigates to the registration page.
The user enters personal details and submits the registration form.
The system validates the data and creates the account.
The user can update their profile and change credentials from their account settings.
Alternative Flow 1: If the registration form has missing or incorrect data:
The system displays an error message ("Username already in use" or "Password too weak").
The user corrected the issue and resubmitted the form.
Post-condition: The user has an account and can access the platform's features.
